# Zotero Duplicate Merge Manager Plugin

一个独立的 Zotero 插件，用于识别并保守处理可能重复的独立条目。

## 功能

- 在一个或多个顶层文献条目右键菜单中添加 `查找可能重复条目...`
- 将选中的多个条目分别作为查询种子，扫描各自当前文库中的顶层文献条目
- 主要按标题完全相同或标题高度相似识别候选重复条目
- 标题完全相同的条目会被强制加入候选，不依赖相似度预筛选
- 标题识别会做全角/半角归一化，并忽略标题内部空格差异
- DOI、年份、作者、条目类型只作为辅助证据，提高或降低置信度
- 扫描时先为当前文库建立标题、DOI、年份、作者索引，再对小候选集评分
- 弹窗按查询种子分组展示候选重复条目、匹配原因、所在文件夹、附件/笔记/标签数量
- 多选时若当前选中项中已有题目完全相同的条目，会在结果顶部单独列出
- 弹窗结果区域使用主内容区纵向滚动条，可上下滚动阅读；表格内部只保留横向滚动
- 候选重复条目的标题、分数、匹配原因使用红色字体标识，便于滚动时快速识别
- 点击 `跳转` 可定位候选条目
- 可在确认后将候选条目合并到查询条目
- 可在确认后合并当前选中项中的同题目组
- 可在确认后把候选条目或同题目组非保留项移入 Zotero 回收站

## 匹配规则

- 标题完全相同或高度相似：最高优先级
- DOI 完全相同：辅助加分，但不会单独覆盖明显不同的标题
- 年份相同、作者重合、条目类型相同：辅助增加置信度
- 年份不同或作者不重合：降低置信度

默认每个查询种子只展示分数不低于 `68` 的候选条目，最多展示 `50` 条。

## 修改操作

- `合并到查询`：保留查询条目，把候选重复条目合并进去
- `合并本组`：对多选中标题完全相同的一组条目，自动推荐资料较完整的条目作为保留项
- `保留并合并`：手动指定同题目组中某一条作为保留项
- `回收` / `回收非保留项`：只移入 Zotero 回收站，不做永久删除
- 合并前会检查同文库、同条目类型、可编辑状态；不满足时会拒绝合并

## 构建 XPI

在插件目录运行：

```bash
bash build.sh
```

构建产物会出现在：

- `dist/zotero-duplicate-merge-manager-plugin-0.5.0.xpi`

## 安装

1. 在 Zotero 中打开 `工具 -> 插件`
2. 点击右上角齿轮
3. 选择 `Install Add-on From File...`
4. 选择构建好的 `.xpi`

## 兼容性

- Zotero 7

当前 manifest 明确标记为兼容 Zotero `7.0.*`。

## 当前行为说明

- 只处理当前文库
- 只扫描顶层文献条目
- 支持多选批量查询；每个分组只排除当前查询条目本身，其他已选条目也可作为候选重复项显示
- 修改操作必须经过确认弹窗
- 回收操作调用 Zotero 回收站，不做永久删除
- 合并调用 Zotero 内置 `Zotero.Items.merge()`，被合并条目会进入回收站
